### Mid-session checkpoint — 2026-05-18 overnight stack

Four more features landed on top of 0.3.0 (still unpublished — per
memory `sdk_version_bump_preference.md`, stack at one version). All
443 SDK tests pass; coverage gate (100/100/100/100) still green.

  - `<SourceViewer>` SDK component (data-agnostic: `source` +
    `highlightLine`, optional gutter + header). 15 tests.
  - `parseEvents(frame, signature)` generic by-signature event filter.
    Accepts either a Solidity event signature string OR a precomputed
    32-byte topic hash. Returns `MatchedEvent[]` (log + logIndex).
    First runtime viem import in the SDK (`toEventSelector`). 13 tests.
  - `TOKEN_SENT_TO_TOKEN_CONTRACT` risk rule + `classifyAddress`
    option on `AnalyzeRisksOptions`. Self-transfers always flagged;
    cross-token (tokenA → tokenB's contract) flagged when the
    classifier returns true. 10 tests.
  - **Graceful shutdown in `packages/api/src/index.ts`** — orchestrator
    closes HTTP server → awaits cache writes → ends pg pool → tears
    down forks, with a 15s hard cap (`setTimeout().unref()`). Plus
    `forkManager.setExitOnSignal(false)` + public `cleanupAll()` so
    the API owns the final `process.exit`.

Next bump (still 0.4.0): adds `<StepDebugger>`, `<SwapsPanel>`,
`<ApprovalsPanel>`, `<SourceViewer>`, `parseEvents`, and
`tokenSentToTokenContract`. All additive.

  consumers who don't use it tree-shake the dep away.

  `setExitOnSignal(false)` is called inside `start()`; the order is

  `[shutdown] clean exit` in stdout.
