### Trace SDK — current state at HEAD (0.3.0)

`packages/sdk` is `@valve-tech/trace-sdk`, **version 0.3.0 in package.json**.

#### Public-facing API surface

**Types** (`src/types.ts`) — additions
- Parser outputs (cumulative): `TokenDelta`, `BalanceDelta`,
  `TokenApproval` (NEW), `Swap` (NEW) discriminated as
  `SwapV1 | SwapV2 | SwapV3`.
- Risk types: `RiskFlag`, `RiskSeverity`, `RiskFlagType` (now includes
  `"LARGE_APPROVAL"`), `AnalyzeRisksOptions` (now includes
  `largeApprovalThreshold?: bigint`).

**Loaders** — unchanged
- `loadTraceFromObject`, `loadTraceFromFile`, `loadTraceFromHash`
- `normalizeCallFrame` / alias `parseCallTrace`, `normalizeStructLogs`

**Traversal** — unchanged
- `walkCallTree`, `flattenCallTree`, `filterByAddress`, `filterBySelector`,
  `findRevertFrame`, `buildGasProfile`

**Parsers** (`src/parsers/`)
- `parseTokenDeltas` (0.1.1)
- `parsePrestateDiff` (0.1.1)
- `parseApprovals` (NEW in 0.3.0) — ERC-20 Approval decoder, filters
  ERC-721 by topic count.
- `parseSwaps` (NEW in 0.3.0) — V1/V2/V3 discriminated union. V1
  dispatches on 4-topic logs and decodes TokenPurchase/EthPurchase,
  normalizing into a `direction: "buyToken" | "sellToken"` field.
  V2 keeps four unsigned amounts; V3 surfaces signed amounts +
  sqrtPriceX96/liquidity/tick pool state.

**Risks** (`src/risks/`)
- `analyzeRisks` (0.2.x design, still in 0.3.0)
- `delegatecallUnrecognized` rule (existing)
- `largeApproval` rule (NEW in 0.3.0) — flags any Approval whose
  value ≥ threshold. Default threshold = `2n ** 256n - 1n` (literal
  unlimited); consumers pass `largeApprovalThreshold` to widen to e.g.
  `2n ** 128n` for phishing-pattern detection.
- **BREAKING in 0.3.0:** `RiskRule` now returns
  `Omit<RiskFlag, "reverted">[]` (array) instead of `... | null` so
  log-based rules can emit multiple findings per frame. Public
  `analyzeRisks` output shape unchanged.

**Components** — unchanged this session
- `<CallTree>`, `<GasFlamegraph>`, `<OpcodeViewer>`, `<FindingsPanel>`,
  `<StateDiffPanel>`, `<FrameDetailPanel>`, pure helpers exported.

**Templates** — unchanged this session
- `<FullDebuggerLayout>`, `<RevertExplainer>`, `<CompactCallSummary>`.

**Hooks** (`src/hooks/`) — NEW subpath in 0.3.0
- `useOpcodeNavigation(steps, { initialIndex? })` — returns
  `{ currentIndex, step, totalSteps, canGoForward, canGoBack,
     goForward, goBack, jumpTo, jumpToNext, nextCall, nextStorage,
     nextLog }`. Resets to `initialIndex` when `steps` identity
  changes. No source-map dependencies.
- Predicates also exported: `isCallOp`, `isStorageOp`, `isLogOp`.
- Types: `OpcodeNavigation`, `UseOpcodeNavigationOptions`.

**Test infrastructure**
- vitest + jsdom + @testing-library/react + @vitest/coverage-v8
- **362 tests pass**; coverage gated at 100/100/100/100.

**Wiring into web** — unchanged (web has not been migrated to consume
0.3.0 yet; uses workspace `*` link, so any local SDK change is live).

### Blockers / gotchas (unchanged)

- Public PulseChain RPC slow for complex traces. Use private RPC in `.env`.
- Source map recompilation downloads solc on first use (~3s), then caches.
- Anvil must be installed (Foundry toolchain) for fork-based simulation.
