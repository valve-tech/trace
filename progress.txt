### Outstanding work — explicit consumer queue (refreshed 2026-05-18 end-of-session)

**1. Replace `node:vm` action executor with proper isolation** (the
   real fix to item #3 above). Three options:
   - `isolated-vm` (npm, native v8 isolate) — gold standard, adds
     native dep
   - `quickjs-emscripten` (separate JS runtime) — slower, no native dep
   - Child process with `--permission` flags + IPC — more code but
     uses only stdlib

**2. Graceful shutdown handler in `packages/api/src/index.ts`**
   - SIGTERM/SIGINT handler that:
     1. Stops accepting new requests
     2. Calls `awaitPendingCacheWrites()` (already exported from tracer.ts)
     3. Calls `pool.end()`
     4. Calls `forkManager.cleanupAll()`
   - Without this, every API restart drops in-flight cache writes
     and leaks Anvil processes.

**3. Bump SDK version + publish** the three components stacked since
   0.3.0 (`<StepDebugger>`, `<SwapsPanel>`, `<ApprovalsPanel>`).
   Additive only; appropriate as `0.4.0` under semver-zero conventions.

**4. `<SourceViewer>` extraction** (deferred from 0.3.0 plan)
   - Data-agnostic: `source: string` + `highlightLine: number`.

**5. Risk analyzer follow-ups**
   - `TOKEN_SENT_TO_TOKEN_CONTRACT` — needs
     `classifyAddress?: (addr) => boolean` in `AnalyzeRisksOptions`.

**6. More parsers**
   - `parseEvents(frame, eventSig)` — generic by-signature filter.
   - UniV4 swap decoder (hook-style pool architecture, deferred).

**7. DX**
   - Docs site (TypeDoc → static).
   - `npm create @valve-tech/trace-app` CodeSandbox template.

**8. Cosmetics / housekeeping**
   - Rename web's `FindingsPanel.tsx` → `SlitherFindingsPanel.tsx`
     (collides with SDK's runtime `FindingsPanel`).
   - `.serena/project.yml` still says `pulsechain-dev-platform`.
   - Bump `actions/checkout@v4` / `actions/setup-node@v4` before
     September 2026 Node-20 removal.

**9. Web-side adoption**
   - `packages/web` uses workspace `*` link, so 0.3.0 surface is live
     locally but has not been refactored to actually consume the new
