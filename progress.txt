### Token charting feature — scoped, blocked on path decision

User requested charting backed by chifra + IndexedDB, with up to 80%
of user storage. Scoping artifact landed at `docs/CHARTING.md` along
with foundational `packages/api/src/services/chifra/{client,cache}.ts`.
Commit `d9064cc`.

**Critical pre-build discovery — chifra.valve.city is minimally
configured for PulseChain:**

```
GET /monitors?chain=pulsechain&list=true
→ 2 monitored addresses:
   - HEX (0x2b591e99…)  — fully indexed, 237MB, 29.6M records
   - 0xa420…             — basically empty, 9 records
```

Every other PulseChain token returns 0 records on chifra `/export`
because chifra has never walked their address. Adding a monitor is a
server-side admin op (`chifra monitor --addrs <X>`) requiring chifra
access, time (walks back to genesis), and disk space.

Also confirmed during probe: chifra `/export` has NO block-range param
— `first_block`/`last_block`/`start_block`/`since`/`block` etc. all
rejected as `Invalid key`. The only position-aware param is
`appearances`, a mutually-exclusive mode flag (not a range filter).

**Three forward paths in `docs/CHARTING.md` §9 — pick one before build
resumes:**

A. Chifra-only with explicit gating (only-HEX-works UX)
B. Chifra + monitor-on-demand backend (job-tracked indexing requests)
C. Pivot v1 data source to BlockScout `tokentx` (works for every
   token day 1; chifra layered in later for HEX-quality depth)

Doc recommends Path C — the IDB cache, chart UI, and storage release
pattern are reusable across data sources. Awaiting user direction.

  `sdk_version_bump_preference.md`), stack and ship together; never
  auto-bump.
- The SDK now has a runtime viem import for the first time
  (`toEventSelector` in `parseEvents`). Previously type-only. If the
  bundle-size budget tightens, `parseEvents` can be the canary —
  consumers who don't use it tree-shake the dep away.
- Graceful shutdown owns the final `process.exit`. ForkManager's
  `setExitOnSignal(false)` is called inside `start()`; the order is
  HTTP close → cache flush → pool end → forks → exit, with a 15s
  force-exit fallback. To test locally: send SIGTERM and observe
  `[shutdown] clean exit` in stdout.

### Blockers / gotchas (unchanged)

- Public PulseChain RPC slow for complex traces. Use private RPC in `.env`.
- Source map recompilation downloads solc on first use (~3s), then caches.
- Anvil must be installed (Foundry toolchain) for fork-based simulation.
