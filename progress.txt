### Trace SDK — design notes

**Goal:** A standalone package (`packages/sdk`, published as `@pulsechain-dev/trace-sdk`)
that lets users load EVM trace data from any source and render or process each piece
independently. No dependency on this platform's API or database — works with any node
that supports debug_traceTransaction.

**Package structure:**
```
packages/sdk/
  src/
    loaders/         # fetch + normalize trace data into canonical types
    traversal/       # recursive utilities for walking / querying the tree
    components/      # headless-styleable React components
    templates/       # pre-composed layouts (drop-in full debugger, compact summary)
    types.ts         # TraceFrame, OpcodeStep, StateDiff, GasProfile (canonical, not wire format)
  index.ts
```

**1. Loaders** — normalize any input into a canonical TraceFrame tree
- `loadTraceFromHash(txHash, rpcUrl)` — calls debug_traceTransaction, normalizes result
- `loadTraceFromObject(rawTrace)` — accepts a callTracer or structLogger payload already in memory
- `loadTraceFromFile(json)` — parse a JSON export (useful for offline analysis)
- All loaders return `Promise<TraceResult>` with the same shape regardless of source

**2. Recursive traversal utilities** — must use an explicit stack, not recursion,
   to handle arbitrarily deep call trees without stack overflow
- `walkCallTree(frame, visitor)` — depth-first walk with enter/exit hooks per frame
- `flattenCallTree(frame)` — returns ordered flat array with depth + index metadata
- `filterByAddress(frames, address)` — extract all calls to/from a specific contract
- `filterBySelector(frames, selector)` — extract calls matching a 4-byte selector
- `findRevertFrame(frames)` — locate the innermost reverting call
- `buildGasProfile(frame)` — aggregate gas by contract + function from the tree
- `computeStateDiffs(beforeState, afterState)` — returns StateDiff[]

**3. Rendering components** — extract existing debugger components, make them
   importable + headless-styleable (className + style props, no hardcoded colors)
- `<CallTree frames={...} onSelect={...} className={...} />`
- `<GasFlamegraph frames={...} />`
- `<OpcodeViewer steps={...} currentStep={...} />`
- `<SourceViewer source={...} highlightLine={...} />`
- `<StateDiffPanel diffs={...} />`
- `<FrameDetailPanel frame={...} />`

**4. Templates** — pre-composed, drop-in layouts
- `<FullDebuggerLayout trace={...} />` — everything, styled, self-contained
- `<CompactCallSummary trace={...} />` — top-level call + gas in one row
- `<RevertExplainer trace={...} />` — surfaces the revert frame + reason string

**5. Canonical types** (in types.ts, not tied to any wire format):
```typescript
interface TraceFrame {
  type: 'CALL' | 'STATICCALL' | 'DELEGATECALL' | 'CREATE' | 'CREATE2'
  from: Address
  to: Address | null
  value: bigint
  gas: bigint
  gasUsed: bigint
  input: Hex
  output: Hex
  error?: string
  revertReason?: string
  depth: number
  children: TraceFrame[]
  // enriched fields (populated by loaders when available)
  functionName?: string
  decodedInput?: DecodedParam[]
  decodedOutput?: DecodedParam[]
  sourceLocation?: { file: string; line: number; column: number }
}
```

**Key design principles:**
- SDK must be usable outside this app with no hidden platform dependency
- Components accept className/style props so consumers can apply their own theme
- Loaders return the same canonical types regardless of input format
- All traversal utilities use explicit stacks — no recursive calls on user data

### Blockers / gotchas

- Public PulseChain RPC (rpc.pulsechain.com) is slow for complex traces (10-15s). Using a
  private/dedicated RPC via PULSECHAIN_RPC_URL env var is strongly recommended.
- Source map recompilation downloads the correct solc binary on first use (~3s), then caches.
  Requires internet access at debug time for contracts not yet in the local cache.
- Anvil must be installed (Foundry toolchain) for fork-based simulation and fork testnets.
  The integration tests skip gracefully if Anvil is not present.
